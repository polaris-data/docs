Most teams do not need every Polaris surface on day one. Start by choosing the format and delivery path that matches your workflow.

## Compare data surfaces

| Surface | Best for | What you get | Tradeoff |
| --- | --- | --- | --- |
| Standardised formats | Cross-venue analytics, features, and ingestion pipelines | One canonical event envelope across venues | You do not keep venue-native field names |
| Raw payloads | Replayability, parser audits, custom normalization | Unmodified venue-native messages | You must handle venue-specific schemas yourself |
| Snapshots | Bulk historical ingestion | Daily `.jsonl.zst` files for standardized datasets | Not the best fit for low-latency recent polling |
| REST endpoints | Recent queries, scripts, product features | Direct API access to trades, events, OHLCV, raw, and metadata | You must page through responses and manage query flow yourself |

## Recommended starting points

### If you are building a research pipeline

Start with [Snapshots](/reference/snapshots) and [Standardised formats](/schemas/standardised-formats). That gives you the fastest path to a local dataset with a stable cross-venue schema.

### If you are validating current market activity

Start with [Trades API](/reference/trades-api) or [Events API](/reference/events-api). Use these when you need recent records rather than bulk historical files.

### If you are building a notebook or chart

Start with [Catalog](/reference/catalog), then choose the query shape that matches your analysis.

- Use [OHLCV API](/reference/ohlcv-api) or `client.ohlcv(...)` for a fast chart or a quick market view.
- Use [Trades API](/reference/trades-api) or `client.trades(...)` when you want execution-level analysis.
- Use [Create a Jupyter notebook with polaris_data](/guides/jupyter-notebook-quickstart) if you want an end-to-end notebook example with pandas and matplotlib.

### If you are debugging venue behavior

Start with [Raw API](/reference/raw-api). Use raw payloads when you need exact venue-native messages or you are maintaining custom parsers.

### If you are unsure what a market supports

Start with [Catalog](/reference/catalog). It tells you which `source` and `market` pairs are valid and shows historical bounds.

## Recommended first path

1. Run the [Quickstart](/quickstart).
2. Confirm access in [Authentication](/guides/authentication).
3. Check market availability in [Catalog](/reference/catalog).
4. Download one file from [Snapshots](/reference/snapshots).
