Use `GET /events` when you want more than just trade executions. This endpoint returns mixed event types in a single normalized stream.

## Endpoint

`GET /events`

## Request shape

| Parameter | Required | Notes |
| --- | --- | --- |
| `exchange` | Yes | Exchange ID |
| `asset` | Yes | Normalized asset or instrument ID |
| `limit` | No | Defaults to `200` without auth and `1000` with auth |
| `cursor` | No | Opaque pagination token |
| `from` | No | Explicit start time; requires bearer auth |
| `to` | No | Explicit end time; requires bearer auth |
| `format` | No | Supports `http` or `file` |

## Parameters

<ParamField query="exchange" type="string" required>

<ParamField query="asset" type="string" required>
  Exact Polaris asset or instrument ID.

<ParamField query="limit" type="number">
  Page size.
</ParamField>

<ParamField query="cursor" type="string">

</ParamField>

  Exclusive UTC end time in ISO 8601 format. Requires bearer auth.
</ParamField>

<ParamField query="format" type="string">

</ParamField>

## Common event types

- `trade`
- `bar`
- `datapoint`
- `orderbook`

## Auth and defaults

| Mode | Behavior |
| --- | --- |
| No bearer auth | `from` and `to` are rejected. The server uses the latest 6-hour window, defaults `limit` to `200`, and caps `limit` at `500`. |
| With bearer auth | `from` and `to` are allowed. If you omit them, the server defaults to a recent 24-hour lookback ending at `now`. Default `limit` is `1000` and max `limit` is `10000`. |

## Response fields

<ResponseField name="format" type="string" required>
  Response mode. JSON responses return `http`.
</ResponseField>

<ResponseField name="data" type="object[]" required>
  Mixed normalized event records such as trades, bars, datapoints, and order book updates.
</ResponseField>

<ResponseField name="next_cursor" type="string">
  Opaque pagination token for the next page, when present.
</ResponseField>

<ResponseField name="has_more" type="boolean" required>
  Whether more results are available.
</ResponseField>

<ResponseField name="limit" type="number" required>
  Effective page size applied to the response.
</ResponseField>

<ResponseField name="total" type="number" required>
  Total number of matching records before pagination.
</ResponseField>

<ResponseField name="range.from" type="string" required>
  Effective UTC start time used for the query.
</ResponseField>

<ResponseField name="range.to" type="string" required>
  Effective UTC end time used for the query.
</ResponseField>

<ResponseField name="range.source" type="string" required>
  `query` when you supplied an explicit range, or `public_window` when the server picked the latest public window.
</ResponseField>

## Python SDK

```python
from datetime import datetime, timedelta, timezone

from polaris_data import PolarisClient

end = datetime.now(timezone.utc)
start = end - timedelta(hours=1)

with PolarisClient() as client:
    events = client.events(
        exchange="hyperliquid",
        asset="SPX",
        from_=start,
        to=end,
    )

print(events[:2])
```
