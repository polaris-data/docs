Use `GET /events` when you want more than just trade executions. This endpoint returns mixed event types in a single normalized stream.

`GET /events` is public for recent data. Send bearer auth only when you need explicit historical bounds or larger result limits.

## Endpoint

`GET /events`

## Request shape

| Parameter | Required | Notes |
| --- | --- | --- |
| `exchange` | Yes | Exchange ID |
| `asset` | Yes | Normalized asset or instrument ID |
| `limit` | No | Defaults to `200` without auth and `1000` with auth |
| `cursor` | No | Opaque pagination token |
| `from` | No | Explicit start time; available with bearer auth |
| `to` | No | Explicit end time; available with bearer auth |
| `format` | No | Supports `http` or `file` |

## Parameters

  Response format. Omit for JSON, or use `file` for a `.jsonl.zst` export redirect.
</ParamField>

## Common event types

- `trade`
- `bar`
- `datapoint`
- `orderbook`

## Auth and defaults

| Mode | Behavior |
| --- | --- |
| No bearer auth | `from` and `to` are rejected. The server uses the latest 6-hour window, defaults `limit` to `200`, and caps `limit` at `500`. |
| With bearer auth | `from` and `to` are allowed. If you omit them, the server defaults to a recent 24-hour lookback ending at `now`. Default `limit` is `1000` and max `limit` is `10000`. |

## Python SDK

```python
from datetime import datetime, timedelta, timezone

from polaris_data import PolarisClient

end = datetime.now(timezone.utc)
start = end - timedelta(hours=1)

with PolarisClient() as client:
    events = client.events(
        exchange="hyperliquid",
        asset="SPX",
        from_=start,
        to=end,
    )

print(events[:2])
```
