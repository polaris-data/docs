Use `GET /ohlcv` when you want interval-based bar data instead of raw executions.

`GET /ohlcv` is public for recent data. Send bearer auth only when you need explicit historical bounds.

## Endpoint

`GET /ohlcv`

## Request shape

| Parameter | Required | Notes |
| --- | --- | --- |
| `exchange` | Yes | Exchange ID |
| `asset` | Yes | Normalized asset or instrument ID |
| `interval` | Yes | Duration token such as `1m`, `5m`, or `1h` |
| `format` | No | Supports `http`, `file`, or `tradingview` |
| `from` | No | Explicit start time; available with bearer auth |
| `to` | No | Explicit end time; available with bearer auth |

## Parameters

<ParamField query="exchange" type="string" required>
  Exchange ID.
</ParamField>

<ParamField query="asset" type="string" required>
  Exact Polaris asset or instrument ID.
</ParamField>

<ParamField query="interval" type="string" required>
  Duration token such as `100ms`, `1s`, `10s`, `1m`, `5m`, `15m`, or `1h`.
</ParamField>

<ParamField query="format" type="string">
  Optional response format. Omit for JSON, use `file` for a `.jsonl.zst` export redirect, or use `tradingview` for TradingView-compatible JSON.
</ParamField>

<ParamField query="from" type="string">
  Inclusive UTC start time in ISO 8601 format. Available with bearer auth.
</ParamField>

<ParamField query="to" type="string">
  Exclusive UTC end time in ISO 8601 format. Available with bearer auth.
</ParamField>

## Auth and defaults

| Mode | Behavior |
| --- | --- |
| No bearer auth | `from` and `to` are rejected. The server uses the latest 6-hour window and returns only the latest 50 bars. |
| With bearer auth | `from` and `to` are allowed. If you omit them, the server defaults to a recent 24-hour lookback ending at `now`. |

## Python SDK

```python
from datetime import datetime, timedelta, timezone

from polaris_data import PolarisClient

end = datetime.now(timezone.utc)
start = end - timedelta(hours=6)

with PolarisClient() as client:
    bars = client.ohlcv(
        exchange="hyperliquid",
        asset="SPX",
        from_=start,
        to=end,
        interval="1m",
    )

print(bars[:2])
```

## File exports

If you set `format=file`, the endpoint returns a `302` redirect to a `.jsonl.zst` export where each line uses the same direct bar shape as the JSON `data` array.
