---
title: "Raw payloads"
description: "Unmodified exchange-native messages for replayability, auditing, and custom parsing."
---

Raw format preserves exchange-native messages as captured, including naming, nesting, and venue-specific field types.

## When raw is useful

- Auditing parser behavior against source messages
- Rebuilding custom normalization pipelines
- Debugging venue schema drift in production

Example:

```json
{
  "timestamp": 1767225600999000,
  "venue": "binance",
  "symbol": "BTCUSDT",
  "type": "raw_trade",
  "data": {
    "e": "trade",
    "p": "64322.10",
    "q": "0.004",
    "m": true
  }
}
```

Raw payload access requires an API key. See [Connecting](/guides/connect) and the [API reference](/reference/api-reference) for request details.
