Raw format preserves venue-native messages as captured, including naming, nesting, and venue-specific field types.

## When raw is useful

- Auditing parser behavior against venue messages
- Rebuilding custom normalization pipelines
- Debugging venue schema drift in production

Example:

```json
{
  "timestamp": 1767225600999000,
  "venue": "binance",
  "symbol": "BTCUSDT",
  "type": "raw_trade",
  "data": {
    "e": "trade",
    "p": "64322.10",
    "q": "0.004",
    "m": true
  }
}
```

Raw payload access requires an API key. See [Authentication](/guides/authentication) and the [Raw API](/reference/raw-api) for request details.
