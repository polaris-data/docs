OHLCV data is interval-aligned and deterministic. Candle boundaries are fixed to UTC intervals, which keeps joins and cross-venue aggregation consistent.

## JSON bar shape from `GET /ohlcv`

- `timestamp`: bar open time in UTC microseconds since Unix epoch
- `open`, `high`, `low`, `close`: interval prices
- `volume`: traded base volume for the bar
- `trades`: number of trades aggregated into the bar
- `interval`: duration token such as `1m`, `5m`, or `1h`

Example:

```json
{
  "timestamp": 1767225660000000,
  "open": 64290.2,
  "high": 64310.5,
  "low": 64271.1,
  "close": 64302.0,
  "volume": 38.47,
  "trades": 124,
  "interval": "1m"
}
```

## TradingView format

If you request `format=tradingview`, the endpoint returns:

- `candles[].time`: Unix seconds
- `candles[].open`, `candles[].high`, `candles[].low`, `candles[].close`
- `volumes[].time`: Unix seconds
- `volumes[].value`: bar volume

## Bar payload inside normalized event streams

When OHLCV appears inside `/events` or snapshot files, it uses the standard event envelope described in [Standardised formats](/schemas/standardised-formats). In that case, the bar-specific fields live under the nested `data` object.
